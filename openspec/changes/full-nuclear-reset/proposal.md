## Why

The MockMate Expo project has accumulated stale Metro caches, corrupted `node_modules`, and bundle-resolution issues causing persistent white-screen errors across iOS and Android. Despite clearing Metro caches and verifying the bundle renders correctly on fresh sims, cached Expo Go instances continue serving broken bundles. A full nuclear reset — wipe the entire `MockMate/` directory and re-scaffold from Expo SDK 54 boilerplate — guarantees a clean foundation with zero accumulated state.

## What Changes

- **BREAKING**: Delete the entire `MockMate/` directory including all source code, dependencies, configs, and platform directories
- Preserve workspace-root files: `AGENTS.md`, `CLAUDE.md`, `DESIGN-MOBILE.md`
- Preserve `llm/` directory (AI prompts, flow docs, Uniwind reference)
- Re-scaffold fresh Expo SDK 54 TypeScript project via `npx create-expo-app@latest MockMate --template blank-typescript`
- Reinstall full stack dependencies: Appwrite (`react-native-appwrite`), RevenueCat (`react-native-purchases`), Sanity (`@sanity/client`, `@sanity/image-url`), AI SDK (`@ai-sdk/openai`, `ai`), Expo AV, Zustand, React Query, Zod, Lucide React Native, Uniwind (Tailwind CSS v4 for RN)
- Reconfigure `app.json` with MockMate scheme, plugins, and Expo SDK 54 settings
- Reconfigure `metro.config.js` with Uniwind integration
- Reconfigure `biome.json` for linting/formatting
- Set up `.env.example` with all required environment variable placeholders
- Set up `global.css` with Uniwind Tailwind v4 imports
- Add `polyfills.ts` with structuredClone and TextEncoderStream/TextDecoderStream polyfills
- Add `dev:clean` npm script for stale cache clearing
- Create minimal `src/app/_layout.tsx` root layout with global.css import
