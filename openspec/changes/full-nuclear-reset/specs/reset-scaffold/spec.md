## ADDED Requirements

### Requirement: Delete MockMate directory

The system SHALL completely remove the `MockMate/` directory and all its contents (source code, dependencies, configs, platform directories, caches) while preserving workspace-root files `AGENTS.md`, `CLAUDE.md`, `DESIGN-MOBILE.md`, and the `llm/` directory.

#### Scenario: Successful deletion

- **WHEN** `rm -rf MockMate/` is executed from workspace root
- **THEN** the `MockMate/` directory no longer exists at the workspace root
- **AND** `AGENTS.md`, `CLAUDE.md`, `DESIGN-MOBILE.md` remain unchanged at workspace root
- **AND** the `llm/` directory and its contents remain unchanged at workspace root

#### Scenario: Preserved files untouched

- **WHEN** deletion completes
- **THEN** `ls AGENTS.md CLAUDE.md DESIGN-MOBILE.md llm/` returns all four items
- **AND** each file's content matches its pre-deletion state

### Requirement: Scaffold fresh Expo SDK 54 TypeScript project

The system SHALL create a new Expo project named `MockMate` at the workspace root using the official Expo SDK 54 TypeScript blank template, resulting in a minimal bootable project.

#### Scenario: Scaffold creates correct directory

- **WHEN** `npx create-expo-app@latest MockMate --template blank-typescript` is executed from workspace root
- **THEN** a `MockMate/` directory exists containing `app.json`, `package.json`, `tsconfig.json`, `app/_layout.tsx`, and `.gitignore`
- **AND** `app.json` has `"sdkVersion": "54.0.0"` or higher in the expo config

#### Scenario: Project boots without errors

- **WHEN** `npx expo start --clear` is executed inside `MockMate/`
- **THEN** Metro bundler starts without fatal errors
- **AND** no red screen or white screen appears in Expo Go when connecting to the Metro URL

### Requirement: Install full stack dependencies

The system SHALL install all required dependencies at latest compatible versions, covering Appwrite auth, RevenueCat payments, Sanity CMS, AI SDK, state management, styling, validation, icons, audio, and testing.

#### Scenario: All dependencies resolve

- **WHEN** `npm install` completes after dependency list is written to `package.json`
- **THEN** `node_modules/` contains all listed dependencies
- **AND** `npm ls` shows no unmet peer dependency errors for production dependencies

#### Scenario: Uniwind works

- **WHEN** `npx expo start --clear` boots the project
- **THEN** `className` props render Tailwind classes correctly in the app
- **AND** `uniwind-types.d.ts` is auto-generated by Metro

### Requirement: Recreate all config files

The system SHALL recreate `metro.config.js` with Uniwind integration, `biome.json` for linting/formatting, `global.css` with Tailwind v4 and Uniwind imports, `polyfills.ts` with structuredClone and TextEncoderStream/TextDecoderStream, `jest.config.js` with Expo preset, `.env.example` with all required placeholder keys, and `.npmrc` with legacy peer deps flag.

#### Scenario: Metro config with Uniwind

- **WHEN** `metro.config.js` is read after setup
- **THEN** it contains `const { withUniwindConfig } = require('uniwind/metro')` and wraps the default config with `withUniwindConfig(config, { cssEntryFile: './global.css', dtsFile: './uniwind-types.d.ts' })`

#### Scenario: Biome config

- **WHEN** `npx @biomejs/biome check --write src/` is executed
- **THEN** it formats without errors
- **AND** `biome.json` exists at `MockMate/` root

#### Scenario: Global CSS imports

- **WHEN** `global.css` is read
- **THEN** it contains `@import 'tailwindcss'` and `@import 'uniwind'`

#### Scenario: Polyfills load

- **WHEN** `polyfills.ts` is imported from `_layout.tsx`
- **THEN** `structuredClone` and `TextEncoderStream`/`TextDecoderStream` are available in the JS runtime

#### Scenario: Environment variable template

- **WHEN** `.env.example` is read
- **THEN** it contains placeholder keys for: `EXPO_PUBLIC_APPWRITE_ENDPOINT`, `EXPO_PUBLIC_APPWRITE_PROJECT_ID`, `EXPO_PUBLIC_APPWRITE_BUCKET_ID`, `APPWRITE_JWT_SECRET`, `EXPO_PUBLIC_REVENUECAT_APPLE_API_KEY`, `EXPO_PUBLIC_REVENUECAT_GOOGLE_API_KEY`, `SANITY_PROJECT_ID`, `SANITY_DATASET`, `SANITY_API_TOKEN`, `OPENAI_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_MODEL`, `OPENAI_TTS_MODEL`, `OPENAI_STT_MODEL`, `EXPO_PUBLIC_DEV_AUTH_BYPASS`

### Requirement: Create minimal root layout

The system SHALL create `src/app/_layout.tsx` that imports `global.css`, polyfills, and provides a skeletal provider chain comment placeholder for future AuthProvider, QueryProvider, and VIPProvider.

#### Scenario: Layout renders without crash

- **WHEN** the app boots in Expo Go
- **THEN** a screen renders (even if blank or with minimal text)
- **AND** no red error screen appears
- **AND** `global.css` styles are applied

#### Scenario: Polyfills imported first

- **WHEN** `_layout.tsx` is read
- **THEN** `import '../../polyfills'` appears before any other import or provider setup

### Requirement: Add dev:clean script

The system SHALL add a `dev:clean` npm script that clears Metro caches, Expo Go shared preferences on iOS simulators, Expo Go app data on Android emulators, and starts Expo with `--clear`.

#### Scenario: Script exists and is executable

- **WHEN** `npm run dev:clean` is invoked from `MockMate/`
- **THEN** the script executes without syntax errors
- **AND** Metro `--clear` flag is passed to `npx expo start`
