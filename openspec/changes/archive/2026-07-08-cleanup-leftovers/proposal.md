## Why

The `app-flow-implementation` branch accumulates five distinct categories of leftover debt from prior partial rewrites (Clerk → Appwrite swap in commit `054de39`, then a screen-level refactor):

1. **Dead Auth0 code**: `MockMate/src/lib/auth/Auth0Provider.tsx` imports a
   package (`react-native-auth0`) that was uninstalled in commit `054de39`,
   breaking Metro bundling (`Unable to resolve "react-native-auth0"`
   raised on Android bundle after 22156ms).
2. **Leaked API key**: `MockMate/.env.example` line 61 ships the real
   `OPENAI_API_KEY=sk-00177295469e3bb9-5e7mt2-6f16cedf` and a private LLM
   proxy URL (`https://9router.chickenkiller.com/v1`). Even though
   `.env.example` is the template, committed-with-a-real-value spreads
   the secret into every clone and fork.
3. **Build red**: `cd MockMate && npx tsc --noEmit` reports 130 errors;
   `npx @biomejs/biome check .` reports 78 errors + 3 infos. Includes 9
   inline `style={{}}` violations of the Uniwind-only rule, missing
   module resolutions, and `xp/gems/streak` shape mismatches in
   `gamificationStore.ts`. **This change does NOT fix build red —
   see "Out of bridge" below.** Build restoration is high-churn
   (rewrites API routes, gamificationStore shape, skeleton types)
   and warrants its own dedicated change. Including it here would
   conflate "deletion cleanup" with "implementation work" and break
   the minimal-diff rule.
4. **Stale Pods cache**: `MockMate/ios/Pods/Local Podspecs/ClerkGoogleSignIn.podspec.json`
   remains on disk even though Clerk packages are gone.
5. **Stale docs**: `MockMate/README.md` and root `README.md` still
   document Auth0 as the auth backend (auth rows, troubleshooting
   sections, `.env.example` snippets).

Each is small individually; they're a single conceptual unit: "the
project does not match its committed documentation and bundling fails on
a brand-checked-out copy."
